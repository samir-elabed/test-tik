🚀 Step 1: Starting the System

First, you need to start the backend and frontend services. There are three easy ways:

Option 1: Windows (Easiest)

# Simply double-click: start.bat

Three windows will open automatically!

Option 2: Linux/Mac

cd tiktok-stream-recorder
./start.sh

Option 3: Docker

cd tiktok-stream-recorder
docker-compose up -d
💡 Pro Tip: Wait 10-30 seconds after starting for all services to initialize properly.

🌐 Step 2: Opening the Web Interface

🔍 Step 4: The Monitoring Process

Once you've added users, the system automatically starts monitoring them. Here's what happens behind the scenes:

1

Check Every 60 Seconds

System checks if monitored users are live

2

Detect Live Stream

When user goes live, detection happens instantly

3

Start Recording

Recording begins automatically using yt-dlp

4

Save & Process

Video saved, thumbnail generated

5

Upload to Telegram

(Optional) Automatically posts to your channel

✨ Completely Automatic! You don't need to do anything. Just add users and the system handles everything else.

📱 Step 7: Telegram Integration (Optional)

The best part! All recordings can automatically upload to your Telegram channel for cloud storage and easy sharing.

How to Enable:

# 1. Create bot with @BotFather
# 2. Create a channel
# 3. Add bot as admin
# 4. Configure .env file:

TELEGRAM_BOT_TOKEN=your_bot_token
TELEGRAM_CHANNEL_ID=@your_channel

💡 Pro Tips:
  • Videos are named: username_timestamp.mp4
  • Each video has a matching .jpg thumbnail
  • Database stores all metadata (duration, title, status)
  • Check logs/ if something goes wrong
  • Backup videos/ folder periodically
